Dotstay - Interactive map

Dotstay - Interactive map

What is Dotstay?

Dotstay is a platform that helps all-around, throughout the relocation journey, anyone who needs to move to a new city. The service is available for the following cities: Milan, Turin, Florence and Rome.

The interactive map

To help you get a complete overview of the city you plan to move to, based on your needs and budget, we decided to develop, in the style of Immobiliare, Booking and similar services, an interactive map:

  • The first step was to find the data of the interested cities. The following repository can be found on GitHub: geojson-italy, containing geojsons with information about the boundaries and areas of major Italian regions and provinces (source: ISTAT), which we then skimmed to make it consistent with the data in our database.

  • As a map, we chose OpenStreetMap, since we preferred to rely on a project made and carried out by the community. For the icons, points of interest, and buttons to interact with the map, we relied on Leaflet. Finally, regarding the map layer and geolocation calls (search by address), we used Mapbox.

Timelinebit - Task Schedule (Standby)

What is Timelinebit?

Timelinebit is a calendar for managing projects and their progress. Currently on hold.

The calendar

The idea was to have a sliding calendar in which all tasks for the selected month could be displayed. In the future there would also be specific calendars depending on the project involved:

  • Notifications are implemented in the following way: when a new task is inserted, I collect the identifiers of all the users assigned to it. Once the task is properly stored in the db, for each user I store a notification record with information about the corresponding task, its creator, its urgency, and the status of the notification, i.e. "to be read." By clicking the circle next to the notification, it is then possible to flag it as "read."

  • This was made with XSLT, HTML, CSS (SASS and Bootstrap) and Javascript (JQuery).

The chat

I wanted to try my hand at the WEB API of EventSources:

  • The initial idea was to use WebSockets, but given the simplicity of EventSources, and since this is not an advanced messaging application, I thought that as a first release it might be okay to use a technology that naturally uses the HTTP protocol.
    In another context I would not adopt this solution, since persistently keeping the server listening for new messages does not seem the most efficient choice.

  • This was made with XSLT, HTML, CSS (SASS and Bootstrap), Javascript (JQuery) and PHP.
